## 2.0.0 - 2026-04-13
* The `BadRequestError.body()` method now returns `Object` instead of `Error`. Callers that previously used the typed `Error` return value will need to cast or update their code accordingly.
* The SDK now supports Company Search mode for job submission: `SubmitRequestDto` has a new optional `connectedDatasetIds` field (and corresponding builder methods) that links dataset IDs to a job, returning only events relevant to companies in those datasets. Additionally, `AsyncMonitorsClient` now exposes `listMonitors()` overloads to retrieve all monitors created by the authenticated user.
* The SDK now exposes two new resource clients: `EntitiesClient` (via `CatchAllApi.entities()`) and `DatasetsClient` (via `CatchAllApi.datasets()`), with async equivalents available on `AsyncCatchAllApi`. The `Record` type gains a new optional `connectedEntities` field (`Optional<List<ConnectedEntity>>`) that is populated when a job is submitted with `connected_dataset_ids`.
* The SDK now includes a new `AsyncDatasetsClient` that provides fully asynchronous (`CompletableFuture`-based) access to the Datasets API. Consumers can list, create, retrieve, update, and delete datasets, manage entities within datasets, and upload CSV files — all without blocking the calling thread.
* The SDK now includes `AsyncRawDatasetsClient`, a fully asynchronous client for managing datasets. All dataset operations (list, create, get, update, delete, CSV upload, entity management, and status history) are available as non-blocking `CompletableFuture`-returning methods, supporting both `File` and `InputStream` inputs for CSV uploads.
* The SDK now includes a `DatasetsClient` that provides full dataset management capabilities. Consumers can create datasets from entity ID lists or CSV uploads, list and update datasets, manage entities within datasets, and retrieve dataset status history. CSV upload methods support both `File` and `InputStream` inputs with optional `MediaType` configuration.
* The SDK now includes a new `RawDatasetsClient` that exposes full dataset management capabilities: create datasets from entity ID lists or CSV files, list, update, and delete datasets, manage entities within datasets, upload CSVs to existing datasets, and retrieve dataset status history. Both `File` and `InputStream` upload variants are supported.
* The SDK now includes a full set of request classes for the datasets resource. New classes — `AddEntitiesToDatasetRequest`, `CreateDatasetRequest`, `CreateDatasetFromCsvRequest`, `DeleteDatasetRequest`, `GetDatasetRequest`, `GetDatasetStatusHistoryRequest`, and `ListDatasetsRequest` — are available in `com.newscatcher.catchall.resources.datasets.requests` and support dataset management operations including creation, retrieval, listing with filters, and deletion.
* The SDK now includes a new `AsyncEntitiesClient` with full asynchronous CRUD support for company entities, including list, create, batch-create, get, delete, and update operations. New dataset request types (`ListEntitiesInDatasetRequest`, `RemoveEntitiesFromDatasetRequest`, `UpdateDatasetRequest`, and `UploadCsvToDatasetRequest`) are also available for managing entities within datasets.
* The SDK now includes a new `AsyncRawEntitiesClient` that provides non-blocking, `CompletableFuture`-based access to the entities API. Consumers can now list, create (single and batch), retrieve, update, and delete entities asynchronously using the new client.
* The SDK now includes an `EntitiesClient` with full CRUD support for company entities. New methods — `listEntities`, `createEntity`, `createEntitiesBatch`, `getEntity`, `deleteEntity`, and `updateEntity` — are available for managing entities within an authenticated organization. A `RawEntitiesClient` is also available via `withRawResponse()` for consumers who need access to HTTP response metadata.
* The SDK now includes new request and model types for entity management. Callers can use `ListEntitiesRequest`, `GetEntityRequest`, `DeleteEntityRequest`, and `UpdateEntityRequest` to interact with the entities resource, along with new `AdditionalAttributes` and `CompanyAttributes` types for rich company-level metadata.
* The SDK now includes new model types for entity management and dataset ingestion: `ConnectedEntity`, `CreateEntityRequest`, `CreateEntityResponse`, `CreateEntitiesBatchResponse`, and `CreateDatasetCsvResponse`. These classes enable consumers to create individual or batches of entities and upload CSV-based datasets via the NewsCatcher API.
* The SDK now includes new types for dataset management: `DatasetResponse`, `DatasetListResponse`, `DatasetEntityListResponse`, and `DatasetEntityIdsRequest`. These classes support listing datasets, retrieving paginated entity summaries, and specifying entity IDs for dataset operations.
* The SDK now includes new types for dataset lifecycle management and entity listing. `DatasetSortBy` and `DatasetStatus` enums have been added alongside `DatasetStatusEntry` and `DatasetStatusHistoryResponse` for tracking dataset status history. A new `EntityListResponse` type is also available for working with paginated entity results.
* The SDK now includes new types for entity management: `EntityResponse`, `EntitySummary`, `EntityType`, `EntityStatus`, and `EntitySortBy`. These types support working with company and person entities, their status lifecycle (`pending`, `enriching`, `ready`, `failed`), and sorting options.
* The SDK now includes new types to support entity management and CSV dataset uploads. `ManageEntitiesResponse`, `UploadCsvToDatasetResponse`, `ValidationReport`, `SkippedRow`, `EntityValidationErrorBody`, and `SortOrder` are now available for use with the corresponding API endpoints.
