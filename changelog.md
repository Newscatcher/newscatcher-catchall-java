## 2.0.0 - 2026-04-13
* The `BadRequestError.body()` method now returns `Object` instead of `Error`. Callers that previously relied on the typed `Error` return value must cast or inspect the body at runtime. No methods have been added or removed from `JobsClient` or `AsyncJobsClient`.
* The SDK now supports **Company Search mode** for batch jobs: `SubmitRequestDto` accepts a new optional `connectedDatasetIds` field that links a job to one or more datasets, causing results to include a `connected_entities` array scored per company. New `listMonitors()` overloads are also available on `AsyncMonitorsClient` to retrieve all monitors created by the authenticated user.
* The SDK now exposes two new resource clients: `EntitiesClient` (via `CatchAllApi.entities()`) and `DatasetsClient` (via `CatchAllApi.datasets()`), along with their async counterparts. The `Record` type gains an optional `connectedEntities` field (`Optional<List<ConnectedEntity>>`) populated when a job is submitted with `connected_dataset_ids`.
* The SDK now includes `AsyncDatasetsClient`, which provides `CompletableFuture`-based async access to the full Datasets API. Consumers can list, create, retrieve, update, and delete datasets, manage entities, upload CSV files, and retrieve dataset status history — all asynchronously.
* The SDK now includes a new `AsyncRawDatasetsClient` class that exposes the full Datasets API via `CompletableFuture`-based asynchronous methods. This includes creating, retrieving, updating, and deleting datasets, managing dataset entities, uploading CSV files, and querying dataset status history — all without blocking the calling thread.
* The SDK now includes a `DatasetsClient` for full dataset lifecycle management. New methods allow creating, retrieving, updating, and deleting datasets, as well as managing entities within datasets via both JSON and CSV upload. Dataset status history can also be retrieved through the new `getDatasetStatusHistory` method.
* The SDK now includes a new `RawDatasetsClient` that provides full access to the datasets API. Consumers can create, retrieve, update, and delete datasets (including CSV-based creation and uploads), manage entities within datasets, and retrieve dataset status history.
* The SDK now includes a full suite of request classes for the Datasets API, including `CreateDatasetRequest`, `ListDatasetsRequest`, `AddEntitiesToDatasetRequest`, `CreateDatasetFromCsvRequest`, `DeleteDatasetRequest`, `GetDatasetRequest`, and `GetDatasetStatusHistoryRequest`. These classes provide strongly-typed, builder-pattern access to all dataset management operations.
* The SDK now includes an `AsyncEntitiesClient` that supports listing, creating (individually or in batch), retrieving, deleting, and updating company entities. Four new dataset request classes — `ListEntitiesInDatasetRequest`, `RemoveEntitiesFromDatasetRequest`, `UpdateDatasetRequest`, and `UploadCsvToDatasetRequest` — have also been added to support expanded dataset management operations.
* The SDK now includes a new `AsyncRawEntitiesClient` that provides asynchronous, non-blocking access to the full entities API. Consumers can list, create (single and batch), retrieve, update, and delete entities via `CompletableFuture`-based methods.
* The SDK now includes a new `EntitiesClient` for managing company entities. Consumers can list, create (individually or in bulk), retrieve, update, and permanently delete entities via the `catchAll/entities` endpoints. A `RawEntitiesClient` is also available for access to raw HTTP response metadata.
* The SDK now includes request and model types for entity management operations. New classes `DeleteEntityRequest`, `GetEntityRequest`, `ListEntitiesRequest`, and `UpdateEntityRequest` are available under the entities resource, along with supporting types `AdditionalAttributes` and `CompanyAttributes` for structured entity data.
* The SDK now includes new types for entity and dataset management: `ConnectedEntity`, `CreateEntityRequest`, `CreateEntityResponse`, `CreateEntitiesBatchResponse`, and `CreateDatasetCsvResponse`. These types support creating and managing custom entity datasets via the NewsCatcher API.
* The SDK now includes new types for dataset management: `DatasetResponse`, `DatasetListResponse`, `DatasetEntityListResponse`, and `DatasetEntityIdsRequest`. These types support listing datasets, browsing entity summaries within a dataset, and specifying entity IDs for add/remove operations.
* The SDK now includes new types for dataset status tracking and entity listing. `DatasetStatus` and `DatasetSortBy` enums, along with `DatasetStatusEntry` and `DatasetStatusHistoryResponse` models, enable consumers to work with dataset lifecycle history. A new `EntityListResponse` type provides paginated access to entity query results.
* The SDK now includes new types for entity management: `EntityResponse`, `EntitySummary`, `EntityType`, `EntityStatus`, and `EntitySortBy`. These types support working with company and person entities, including their full lifecycle statuses (`PENDING`, `ENRICHING`, `READY`, `FAILED`) and sorting options.
* The SDK now includes new types for entity dataset management and CSV uploads: `EntityValidationErrorBody`, `ManageEntitiesResponse`, `UploadCsvToDatasetResponse`, `ValidationReport`, `SkippedRow`, and a `SortOrder` enum. These types support entity add/remove operations and CSV-based bulk entity creation with per-row validation reporting.
